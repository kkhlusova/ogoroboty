# OmegaBot

Платформа дистанционного управления мобильным роботом с пультом оператора,
видеопотоком, телеметрией, логированием и откатом движения при потере связи.

## 1. О проекте

Управление OmegaBot — это распределённая система для дистанционного управления мобильным роботом. Проект предназначен для работы в связке «Оператор — Робот», где управление осуществляется через графический интерфейс (GUI) на компьютере оператора, а исполнение команд происходит на бортовом компьютере робота (Raspberry Pi3) и микроконтроллере (Arduino).
    
Система состоит из трёх основных частей:
    1. Пульт оператора (Operator Console): Qt-приложение (C++), предоставляющее интерфейс управления, приём видео и обработку данных.
    2. Бортовой компьютер робота (Robot Server): Linux-приложение (C++), принимающее команды через SSH/сеть, управляющее камерой и передающее данные с микроконтроллера.
    3. Микроконтроллер (Arduino): Прошивка (new_pisk.ino), управляющая моторами, сервоприводами и считывающая показания датчиков.

## 2. Основное назначение

- Дистанционное управление движением робота (вперёд, назад, повороты);
- Получение видеопотока в реальном времени с детекцией объектов (YOLO);
- отображение телеметрии: ИК-датчик препятствий;
- локальное сохранение кадров и сборка видео на стороне оператора;
- логирование команд на стороне оператора и робота;
- Выполнение пакетных команд (скриптов движения).
- heartbeat-контроль связи между оператором и роботом;

## 4. Минимальный сценарий запуска

```bash
# На пульте оператора
cd robot
mkdir build && cd build
cmake ..
make
./mainwindow
``` 
# На роботе
Включить устройство
Настроить подключение к одной сети wi-fi робота и ноутбука оператора
```

После запуска:
1. нажмите `Connect`;
2. проверьте, что статус стал `Connected`;
4. начните управление с клавиатуры `w/a/s/d/q/e` и `i/j/k/l`.

## 5. Требования и сборка

Все представленные примеры приведены для Ubuntu/Debian-подобных систем.

### Базовые инструменты

```bash
sudo apt update
sudo apt install -y build-essential cmake pkg-config git
```

### Для `robot/`

- компилятор C++20
- OpenCV
- доступ к:
  - камере OpenCV (`camera_index = 0` по умолчанию),
  - последовательному порту `/dev/ttyACM0`

```bash
sudo apt install -y libopencv-dev
```

### Для `operator/`

- компилятор C++20
- Qt5 с модулем `Widgets`
- OpenCV
- утилиты командной строки:
  - `ffmpeg` - для сборки видео из кадров;

```bash
sudo apt install -y \
    qtbase5-dev \
    libopencv-dev \
    ffmpeg \
    libavformat-dev \
    libavcodec-dev \
    libavutil-dev \
    libswscale-dev
```

## 6. Управление в GUI

### Движение робота

| Клавиша | Действие |
|---|---|
| `w` | вперёд |
| `s` | назад |
| `a` | влево |
| `d` | вправо |
| отпускание клавиши | стоп |

### Управление видеопотоком

| Кнопка / клавиша | Действие |
|---|---|
| `High quality` / `x` | увеличивает частоту отправки кадров |
| `Low quality` / `x` | уменьшает частоту отправки кадров |
| `Start video` | начинает запись видео |
| `Stop video` | останавливает запись и сохраняет ее на ноутбуке оператора|

### Пакетные команды

В блоке пакетных команд можно сформировать список команд вручную.

Пример ввода команд:

```
f 1000
left 20000
r 15000
```

После нажатия `Comply` робот выполняет написанные команды в порядке очередности.

## 7. Хранение получаемых данных

### На пульте оператора

Каталог:

```text
~/robot_info/
```

Содержимое:

- `robot.log` - лог команд и системных событий;
- `YYYMMDD_HHMMSS.avi` - сохраненная запись видео;
- /yolo - файлы модели YOLO

### На стороне робота

Каталог:

```text
~/robot_video/
```

Содержимое:

- `commands.log` - лог команд и системных событий;
- `sensors.log` - лог телеметрии;

## 9. Подготовка окружения

### Подготовка YOLO-файлов на пульте оператора

Текущая реализация ожидает наличие каталога:

```text
~/robot_info/yolo
```

Внутри должны лежать файлы:

```text
~/CFG_YOLO/yolov3-tiny.cfg
~/CFG_YOLO/yolov3-tiny.weights
~/CFG_YOLO/coco.names
```

Если эти файлы отсутствуют, приложение продолжит работу, но YOLO-детекция будет не активна.

### Подключение аппаратуры на стороне робота

Ожидается, что:

- камера доступна с индексом `0`;
- робот и оператор находятся в одной сети и нужные порты(5000) не заняты.

## 10. Сетевые интерфейсы и протоколы

###Передача видео

Порт - 5000
Протокол передачи видео - UDP
Формат - JPEG

###Связь

Протокол - SSH
Таймаут подключения - 5 секунд
Порт - 22
Режим - BatchMode=yes (без пароля)
Keep-alive - ServerAliveInterval=30
